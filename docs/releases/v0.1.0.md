# MaintainerBench v0.1.0 Release Notes

MaintainerBench v0.1.0 is the first public release for maintainers who want practical guardrails around terminal-based AI coding agents.

## Highlights

- Initialize a repository with `AGENTS.md`, repo-local skills, MaintainerBench config, an example task, and a CI workflow.
- Lint AI-agent workflow files for missing guidance and unsafe patterns.
- Define benchmark tasks in YAML.
- Run an external agent command in a temporary git worktree.
- Run setup and verification commands.
- Analyze changed files and diff content for basic risk rules.
- Write maintainer-readable Markdown reports and stable JSON reports.
- Run lint checks from the MaintainerBench GitHub Action.
- Publish `@maintainerbench/cli` through a manual GitHub Actions release workflow.
- Try the TypeScript and Python examples without requiring Codex or any other model-backed agent.

## Quick Trial

```bash
pnpm install
pnpm build
pnpm exec maintainerbench lint
pnpm exec maintainerbench eval \
  --task examples/ts-library/.maintainerbench/tasks/add-multiply-helper.yml \
  --agent-command "node examples/fake-agents/add-ts-multiply.mjs"
```

## What Is Intentionally Limited

- No web app, SaaS backend, database, auth system, telemetry, or model API calls.
- No automatic approval, merge, push, deploy, publish, or pull request acceptance from MaintainerBench runtime commands.
- GitHub Action eval mode is not supported in v0.1.0.
- Eval worktrees are useful isolation for repository diffs, but they are not a full security sandbox.

## Publishing

Use the `Release` workflow from GitHub Actions. The workflow defaults to `dry_run: true`; set `dry_run: false` only when publishing `@maintainerbench/cli` to npm. Publishing requires the `NPM_TOKEN` repository secret.

## For Maintainers

Use v0.1.0 to start documenting agent expectations, lint repository workflow files, and benchmark small coding-agent tasks. Treat reports as review aids, not as proof that a change is correct or secure.
